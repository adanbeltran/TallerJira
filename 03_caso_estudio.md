En esta sección se presentan las acciones mínimas necesarias para desarrollar el taller.

## 2.1 Crear cuenta e ingresar

Cada participante debe:

1. crear su cuenta en Jira / Atlassian;
2. ingresar a la plataforma;
3. familiarizarse con el menú principal.

## 2.2 Crear un proyecto en Jira

### Paso a paso
1. Ingresar a Jira.
2. Ir a **Projects**.
3. Seleccionar **Create project**.
4. Escoger la plantilla **Scrum**.
5. Asignar nombre al proyecto.
6. Crear el proyecto.

### Nombres sugeridos
- **UniCuídate – Individual**
- **UniCuídate – Equipo X**

## 2.3 Crear y usar el Backlog

El **Backlog** es la lista de trabajo pendiente.

### En el Backlog se debe:
- crear épicas;
- crear historias de usuario;
- ordenarlas por prioridad;
- preparar el trabajo que irá al sprint.

### Paso a paso
1. Entrar al proyecto.
2. Abrir la opción **Backlog**.
3. Crear las épicas.
4. Crear las historias.
5. Asociar cada historia a una épica.
6. Ordenarlas según prioridad.

## 2.4 Crear historias de usuario

Cada historia debe redactarse así:

**Como** [rol]  
**quiero** [necesidad]  
**para** [beneficio]

### Ejemplo
Como estudiante  
quiero solicitar apoyo  
para recibir orientación sin iniciar una denuncia formal.

### Cada historia debe incluir:
- título;
- descripción;
- prioridad;
- criterios de aceptación;
- relación con una épica.

## 2.5 Agregar criterios de aceptación

Cada historia debe tener mínimo **3 criterios de aceptación**.

### Formato sugerido
- Dado...
- Cuando...
- Entonces...

### Ejemplo
Dado que el estudiante completa el formulario,  
cuando envía la solicitud,  
entonces el sistema crea un caso con ID y estado “Recibido”.

## 2.6 Crear un Sprint

El Sprint es el conjunto de historias que se van a trabajar.

### Paso a paso
1. Ir al **Backlog**.
2. Seleccionar **Create Sprint**.
3. Asignar nombre al Sprint.
4. Definir el **Sprint Goal**.
5. Mover historias al Sprint.

### Ejemplo de Sprint Goal
“Permitir que un estudiante solicite apoyo y que Bienestar pueda revisar y dar una primera respuesta.”

## 2.7 Usar el Board

El **Board** muestra el trabajo en curso.

### Estados mínimos
- To Do
- In Progress
- Done

### En el Board se debe:
- mover historias o tareas;
- ver avance;
- identificar bloqueos;
- hacer seguimiento del sprint.

### Paso a paso
1. Abrir el proyecto.
2. Entrar a **Board**.
3. Ver las tareas del Sprint.
4. Moverlas según el estado real del trabajo.

## 2.8 Crear subtareas

Las subtareas permiten dividir una historia.

### Ejemplo
Historia: Solicitar apoyo

Subtareas:
- diseñar formulario;
- validar campos;
- registrar número de caso;
- diseñar mensaje de confirmación.

## 2.9 Registrar comentarios y seguimiento

Jira también permite dejar evidencia del proceso mediante:

- comentarios;
- cambios de estado;
- asignación de responsables;
- tareas terminadas.
